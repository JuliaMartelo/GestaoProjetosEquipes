# 📌 Sistema de Gestão de Projetos e Equipes

## 📖 Introdução

A Oracle possui diversos clientes cujas demandas de trabalho são totalmente orientadas a projetos. De modo geral, observa-se a necessidade de um controle mais eficiente desses projetos, cujo desenvolvimento é de responsabilidade de colaboradores que desempenham funções como desenvolvedor, analista de sistemas e designer de interface.

Esse controle envolve a distribuição das etapas do projeto entre os colaboradores, visando um melhor aproveitamento da equipe, bem como o acompanhamento do andamento das atividades, garantindo o cumprimento dos prazos estabelecidos.

Com o aumento do portfólio de projetos, surge uma maior dificuldade na gestão e organização das atividades, bem como na distribuição das tarefas entre os colaboradores. Dessa forma, torna-se necessária a utilização de uma ferramenta adequada e específica para manter o controle do andamento dos projetos, assegurando o cumprimento dos prazos e uma melhor utilização dos recursos humanos disponíveis.

---

## 📌 A Demanda

A principal necessidade identificada consiste no desenvolvimento de um sistema de gestão de projetos e equipes, com o objetivo de proporcionar maior eficiência no controle das atividades.

O sistema deverá permitir o gerenciamento de projetos, incluindo suas tarefas, equipes envolvidas, prazos de entrega e entregáveis. Além disso, deverá possibilitar a geração de relatórios de desempenho para cada projeto, contribuindo para uma melhor tomada de decisão e organização dos processos.

---

## 🎯 Objetivo

Desenvolver um sistema capaz de gerenciar projetos e equipes, permitindo o controle de tarefas, prazos e desempenho, visando otimizar a organização e a produtividade dos colaboradores.

---

## 🛠️ Tecnologias Utilizadas

- Java
- JDBC
- MySQL
- Arquitetura MVC

---

## ⚙️ Funcionalidades

- Cadastro de usuários
- Cadastro de projetos
- Cadastro de equipes
- Listagem de dados
- Atualização de informações
  
---

## 🧱 Estrutura do Projeto

O sistema foi desenvolvido utilizando o padrão MVC:

- **Model:** Representação das entidades (Usuário, Projeto, Equipe)
- **View:** Interface de interação com o usuário
- **Controller:** Regras de negócio
- **DAO:** Comunicação com o banco de dados

---

## 🚧 Desafios

Durante o desenvolvimento, foram enfrentados desafios como:

- Integração com banco de dados (JDBC)
- Organização das entidades e seus relacionamentos
- Garantia do funcionamento correto das operações CRUD
